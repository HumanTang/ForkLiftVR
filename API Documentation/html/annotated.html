AttributesThe attributes are assigned to members of a table definition c# class for each table you wish to query. For example, if you are querying a table of Weapons and the query has the fields WeaponID, WeaponName, and Damage, you would specify this with the following table definition class:

AutoIncrementAttributeAttribute that specifies that the given member is a field that SQLite will auto-increment when inserting. Ex:

CollationAttributeAttribute that defines a collating sequence used for text entries in that column. The default collating sequence is the collating sequence defined for that column in the CREATE TABLE statement. Or if no collating sequence is otherwise defined, the built-in BINARY collating sequence is used. It is the way the SQL engine orders data internally.

DefaultAttributeAttribute that specifies the default value for a field for a given member. The default takes on the type of the member it defines. Ex:

IgnoreAttributeAttribute that specifies that the given member is a field will not be used in the final table mapping. These fields can be extra fields that are calculated, but still need reflection for your other code logic. Ex:

IndexedAttributeAttribute that specifies that the given member is a field that SQLite will index. Ex:

MaxLengthAttributeAttribute that specifies the maximum field size for a member, useful for strings that need to be limited to a given number of characters. Ex:

NotNullAttributeAttribute that specifies that the field for this member cannot be null in the database. Ex:
