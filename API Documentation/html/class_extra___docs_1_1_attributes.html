Detailed Description

The attributes are assigned to members of a table definition c# class for each table you wish to query. For example, if you are querying a table of Weapons and the query has the fields WeaponID, WeaponName, and Damage, you would specify this with the following table definition class:

using SimpleSQL;

public class Weapon
{
// The WeaponID field is set as the primary key in the SQLite database,
// so we reflect that here with the PrimaryKey attribute
[PrimaryKey]
public int WeaponID { get; set; }
public string WeaponName { get; set; }
public float Damage { get; set; }
}

Multiple attributes can be assigned to a single member.
Ex:

using SimpleSQL;

public class StarShip
{
// The starship name will have an index created in the database.
// It's max length is set to 60 characters.
// The name cannot be null.
[Indexed, MaxLength(60), NotNull]
public string StarShipName { get; set; }
}

Note: the table definition does not need to match the table exactly, just the query of the table. If, for instance, you will be querying a joining of two or more tables, your definition class will need to have members for each of the fields present in the query.
